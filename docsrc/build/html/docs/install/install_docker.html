Installing Docker¶

Important

In most cases, Developers will want to leverage the CUDA-capable or ROCm Dockers as they support accelerated quantization. Before installing Docker for CUDA-capable GPUs, ensure that you understand the NVIDIA driver, CUDA Host System Requirements for Vitis AI.

Note

For ROCm distributions, developers should reference ROCm docker installation for further details of docker installation.

Installing NVIDIA Container Toolkit¶

If you are building the Vitis AI Docker Image with CUDA-capable GPU acceleration, you must install the NVIDIA Container Toolkit, which enables GPU support inside the Docker container. Please refer to the official NVIDIA documentation for additional information.

For Ubuntu distributions, NVIDIA driver and Container Toolkit installation can generally be accomplished as displayed in the following example (use sudo for non-root users):

apt purge nvidia* libnvidia*
apt install nvidia-driver-xxx
apt install nvidia-container-toolkit

Where xxx is the version of driver that you are choosing to install (i.e, nvidia-driver-510), and is a version that meets Vitis AI Host System Requirements.

A simple test to confirm driver installation is to run nvidia-smi:

nvidia-smi

The output should appear similar to this:

Docker Install¶

Once you are confident that your system meets any pre-requisites for Vitis AI Docker CUDA or ROCm GPU support, refer to official Docker documentation to install the Docker engine.
